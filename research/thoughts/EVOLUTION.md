## Thinking Modules: Cognitive Architectures

### 1. CoT: Chain-of-Thought (線性序列推理)

* **Sequential-Thinking**：引導模型將複雜問題拆解為線性、連續的邏輯步驟。(推理始祖)
* **Self-Explain**：在輸出答案前，強制執行內部的逐步解釋，以減少邏輯跳躍。
* **Step-by-Step Verification**：針對序列中的每一步進行即時驗證，確保推導基礎穩固。

### 2. ToT: Tree-of-Thoughts (多路徑分支搜索)

* **Brainstorming / Seed Generation**：針對核心問題產生多個並行的潛在解決路徑（思考分支）。  
* **Heuristic Evaluation**：對各個分支的「可行性」進行預判與評分。  
* **Backtracking**：當某一分枝進入邏輯死胡同或不符目標時，回溯至前一節點重新探索。

### 3. GoT: Graph-of-Thought (非線性拓撲推理)

* **Finite-State-Machine (FSM)**：將推理過程定義為不同的狀態轉移，確保邏輯流轉符合預設規則。  
* **Aggregation & Transformation**：允許將多個思考路徑的優點進行「合併（Combining）」或將現有思考點「轉換（Transforming）」為新節點。  
* **Reflection & Self-Correction**：引入回饋迴路，透過批判性審視（Critique）對既有圖譜節點進行修正或修剪。

### 4. AoT: Algorithm of Thoughts (算法化思維)

* **DFS/BFS Integration**：在提示詞中嵌入深度優先或廣度優先搜索的邏輯框架。  
* **Dynamic Planning**：根據中間結果動態調整後續的思考路徑。

## 定義推理方向

以圓的概念說明推理方向(象徵性的敘述風格)

* 徑向 => DENSITY
* 切向 => DIVERSITY 或 給予發揮空間
* 徑向向量的長度增加意義 => 包含面積增加 => DIVERGENCE => 同領域深化
* 切向向量的角度增加意義(規格條件的限制、目的脈絡的感知) => 包含面積增加 => DIVERGENCE => 跨領域擴展
